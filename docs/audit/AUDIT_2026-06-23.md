## 2. Quant Research Integrity — CRITICAL FAILURE

### CRIT-1: PurgedWalkForwardFolds — No cross-fold purging (86% leakage)
**`labels/compat.py:113-131`**
```python
train_idx = idx[:test_end - gap]  # NO cross-fold purging
```
**Impact**: Fold N-1 test data leaks into fold N training set. With `n=848, n_folds=5, gap=20`: **121 of 141 fold-1 test observations (86%) leak into fold-2 training**. Every walk-forward result in the repository is contaminated. All downstream consumers: `walk_forward_backtest.py`, `backtest_pnl.py`, `filter_direction.py`, `retrain_counterfactual.py`.

**Severity**: Critical | **Fix**: Add purging step | **Effort**: 2h

### CRIT-2: Random train_test_split for time-series
**`training.py:134-144`**
```python
x_tr, x_ev, y_tr, y_ev = train_test_split(x_binary, y_vals, test_size=0.2, random_state=42)
```
**Impact**: Triple-barrier labels use prices from `i+1` to `i+20`. Random split assigns training rows whose label windows extend into validation row date ranges. Label lookahead. Metrics are not reliable OOS. Model trained on 80% only.

**Severity**: Critical | **Fix**: Time-series CV | **Effort**: 1h

### CRIT-3: Meta-model on in-sample predictions
**`training.py:190-200`**
```python
primary_pred = model.predict_proba(x_binary)  # IN-SAMPLE
asset._meta_label_model.train(full_train, primary_pred, ...)
```
**Impact**: Meta-model learns that primary model's high-confidence predictions are reliable — but those confidence levels are inflated by in-sample overfitting. Guarantees OOS degradation.

**Severity**: Critical | **Fix**: Train on OOS walk-forward predictions | **Effort**: 4h

### CRIT-4: Expanding window never drops old data
**`training.py:114-116`**: `start_date = end_date - 5 years`. Fallback to ALL data if <200 rows.
**Severity**: Critical | **Fix**: Rolling window | **Effort**: 2h

### CRIT-5: No early stopping
**`training.py:152-163`**: 300 trees trained regardless of validation loss.
**Severity**: Critical | **Fix**: `early_stopping_rounds=50` | **Effort**: 30min

### HIGH Findings

| ID | Issue | File:Line | Effort |
|----|-------|-----------|--------|
| H1 | Production (random split) ≠ backtest (walk-forward) methodology | `training.py:138` vs `walk_forward_backtest.py:185` | 4h |
| H2 | COT features forward-filled across fold boundaries | `alpha_features.py:289-294` | 1h |
| H3 | Zero transaction costs in all backtests | `backtest_pnl.py:50-74` | 2h |
| H4 | yfinance for training labels, MT5 for live execution | `training.py:76` vs `pipeline.py:135` | 3h |
| H5 | SELL_ONLY not applied in walk-forward | `backtest_pnl.py:423` vs `walk_forward_backtest.py` | 1h |
| H6 | Meta-model saved with pickle (fragile) | `meta_labels.py:152-153` | 30m |
| H7 | Macro intersection silently drops date ranges | `data_fetch.py:372-396` | 1h |
| H8 | Portfolio Sharpe = artifact of equal-weight R-multiple averaging | `backtest_pnl.py:235` | 1h |

### Verdict

**The walk-forward backtest results CANNOT be trusted.** The 86% fold-overlap means reported OOS metrics are partially in-sample. The portfolio Sharpe of 29.0 is an artifact of leaky folds + equal-weight averaging + zero transaction costs + zero slippage. Realizable live Sharpe is likely below 2.0.

## 3. Determinism & Reproducibility — Score 6/10

### Can a trading day be reconstructed from persisted state?

**Partially.** The WAL captures the decision chain (`features_snapshot → inference_output → decision_output`) with hash verification. But the following are **missing**:

| Missing Component | Impact | Why Needed |
|------------------|--------|------------|
| Market data WAL | Cannot replay price-dependent SL/TP hits from raw data | No tick/bar-level OHLCV recording |
| Broker response WAL | Cannot reconstruct MT5 fill prices or partial fills | No WAL of execution confirmations |
| RNG seeding per-cycle | Non-deterministic across Python versions | NumPy/XGBoost internal RNG not seeded per-cycle |
| Per-cycle position snapshots | No rollback point | `state_committed` captures only summary |

### Fixes for Full Determinism
1. Record OHLCV price at cycle start alongside `features_snapshot`
2. Log MT5 fill confirmations, SL/TP hits, order rejections to WAL
3. Seed `numpy.random` + `random` with deterministic `(cycle_id, asset_name)` hash
4. Save full position state after each cycle
5. Automate `test_replay_determinism.py` as CI gate

## 4. Trading Engine Review

### Engine Trust Score: 6.5/10
### Production Readiness Score: 6.5/10

### CRITICAL Engine Bugs

#### E1: Consecutive-loss breaker records intra-cycle PnL
**`orchestrator/engine.py:270-275`**
```python
if total_value < prev_value:
    self._circuit_breaker.record_daily_pnl(total_value - prev_value)  # intra-cycle!
```
**Impact**: At 30s cadence, a 3.5-minute drawdown (7 cycles) trips `max_consecutive_losses=7` and **halts the entire portfolio**. Will trigger during normal trading multiple times per day.
**Severity**: Critical | **Fix**: Add date guard (one PnL record per trading day) | **Effort**: 15min

#### E2: Peak portfolio value from two inconsistent sources
**`orchestrator/engine.py:148-149` vs `:225-227`**
Line 148: `self._peak_portfolio_value = total_equity` (mtm_value-based)
Line 227: `self._peak_portfolio_value = max(..., total_value)` (current_value-based)
**Impact**: If `mtm_value > current_value` (floating gains > realized), peak inflates. Drawdown check sees false drawdown permanently — **phantom drawdown that never resets**.
**Severity**: High | **Fix**: Single value source | **Effort**: 30min

#### E3: MT5 mode zero fill simulation
**`bridge.py:108-111, 148-149, 169-170`**
```python
if self._is_real_broker:
    return mid_price, 0.0, 0.0  # no slippage, no impact
```
**Impact**: Paper PnL tracks ideal prices. Real MT5 PnL includes spreads + commissions + slippage. Dashboard shows paper returns, not real returns.
**Severity**: High | **Fix**: Apply basic slippage model | **Effort**: 1h

#### E4: MT5 sizing bypasses leverage budget
**`entry_service.py:522-530`**: MT5 order path skips atomic leverage decrement.
**Severity**: High | **Fix**: Add lock bypass | **Effort**: 2-3h

#### E5: Exposure multiplier applied one cycle late
**`orchestrator/engine.py:249-252`**: Phase 3 sets multiplier after Phase 1 entries already fired.
**Severity**: Medium | **Fix**: Move to pre-Phase 1 | **Effort**: 10min

## 10. Security Audit — Score 5/10

### CRITICAL Vulnerabilities

#### S1: MT5 password cleartext over TCP
**`mt5_client.py:304`** — password sent as cleartext in JSON frames over TCP socket (port 9879). Any process on the same machine or same subnet can sniff the real-money account credential.
**Fix**: TLS or pre-shared key encryption | **Effort**: 4h

#### S2: `pickle.load()` on untrusted file path
**`simulation_snapshot.py:186`** — `pickle.load(f)` on `models/` directory. Arbitrary code execution if attacker can write a pickle file.
**Fix**: Replace with JSON or hash verification | **Effort**: 2h

## 15. Final Verdict

### Strengths
1. **Extraordinary documentation discipline**: AGENTS.md and LIVE_CONTRACT.md are exceptional — every bug, every falsified hypothesis, every fix with dates and line numbers.
2. **Well-designed decision pipeline**: 22-stage composable pipeline on shared `DecisionContext` is a genuinely good architecture.
3. **Comprehensive risk governance**: 14 layers covering regime, stability, meta-labeling, macro, liquidity, PSI drift, filter, calibration, kelly, factor model, cluster alarm, drawdown, circuit breaker, slippage.
4. **Write-ahead log with causal chain**: Feature hash → inference → decision with cross-log integrity verification.
5. **Self-awareness of limitations**: BUY inversion root cause unknown explicitly documented. R-multiple Sharpe artifact acknowledged.

### Critical Risks (Block Institutional Capital)
1. **Walk-forward 86% data leakage** — all research results partially in-sample
2. **BUY inversion root cause unknown** for 8/19 assets — no institutional investor accepts this
3. **MT5 password cleartext on TCP** — SOC2/SOX compliance failure
4. **5 critical engine bugs** — false halts guaranteed, phantom drawdown, MT5 bypasses leverage, no fill simulation
5. **Production ≠ backtest methodology** — research results don't predict production performance

### Immediate Priorities (7 Days, ~10h)
1. Fix PurgedWalkForwardFolds cross-fold purging (2h)
2. Fix consecutive-loss breaker date guard (15m)
3. Fix peak portfolio value single source (30m)
4. Encrypt MT5 bridge TCP (4h)
5. Replace pickle.load() with JSON (2h)
6. Add frontend auth header (1h)
